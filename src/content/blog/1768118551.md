# わたしが Neovim を使えるようになったわけ

挫折を重ねていました。Neovim自体には1年以上前から興味を持っていて、何度か移行しようとしましたがこれまで何度も失敗していました。Neovimの画面を開くもののわからないことが多すぎて、結局VSCodeに戻る日々が続いていました。

ただ、何度目の挑戦でしょうか。4回目くらい？ついに普段の開発で使うエディタをNeovimにすることに成功しました。少しずつオペレータの考え方にも慣れてきたと思います。

このブログでは、わたしがなぜNeovim移行に挫折していたのか、はたまたなぜ成功したのか、振り返ってみます。

## なぜ Neovim を使いたかったのか

これまでPHP、Javaを書いていたときには主に Eclipse(Atom 使ったり、Sublime Textを使ったこともあったか)、現職でPerl、TypeScriptを書くようになってからはVSCodeを使っていました。それで不自由はしておらず、特に最近ずっと使っている VSCodeはプラグインの追加も簡単で困っていることはそこまでありませんでした。

それなのになぜ Neovim を使いたかったのか。

「単純にかっこいいから」です。

YouTubeなどでも vimmer のコーディング動画を見ていると、テキスト編集が圧倒的な速さです。速いのはかっこいいです。

Vim でもよかったのですが、プログラムを書いてテストして、というような開発プロセスを回すにはNeovimのほうが対応しやすいのかな、というイメージでNeovimを選択しました。

実際使ってみてどうだったのかというところはまた後述します。

## なぜ挫折していたのか

Vim にしても Neovim にしても、標準的な機能に加えてプラグインを追加して自分のエディタにできるというのが愛好家が愛好するところのひとつかと思いますが、初心者からみてNeovimを設定しようとしたとき、エディタの標準的な機能なのか、それともプラグインなのか、というところのわかりづらさがありました。

公式ドキュメントを読め、それはそう、という感じなんですが自分にはハードルが高く感じました。

ヘルプを使え、それもそう...

ただ、VSCodeでの設定のわかりやすさ、ワンクリックでいれられる拡張機能になれていた私にはハードルが高かったというところです。

「こういう風な機能をいれたいな」と思ったときに、検索するには言葉を知っていなければならないですし、プラグイン自体も設定項目がたくさんあります。

こうしたハードルを下げてくれたのは Claude Code でした

## なぜ挫折していないのか

結論としては、Claude Code のおかげです。Claude Code が設定を一緒に行ってくれ、「こういうことがしたい」を投げると今の設定を見て返答してくれます。そのため、「この動きはコマンドで便利にできないだろうか」とか「あのコマンド忘れちゃったな」というときに気軽に確認することができ、自分のやりたいことベースで使い方を身につけていくことができたと思います。

Claude Code と一緒にやり始めたきっかけは上司（WezTerm、Emacs使い）がClaude Codeにdotfilesを作らせていたのをみたことです。Claude Codeはローカルファイルを読んでくれるので、いい感じに作ってもらってるよと聞いてその方式でやってみることにしました。

パッケージマネージャーをどうするかなどのベースは kawarimidoll さんの Zenn Book を参考にして、mini.nvim を使うことにしました。最初の導入はClaude Codeと一緒にやってもよいとして、そのあとは自分でメンテナンスしていくということを考えたときに、なんとなくほかよりシンプルに見えたからです。
結果としていまNeovimには挫折しておらず、マイペースにコマンドが身についていっています。ノーマルモードでのカーソル移動含め、テキスト編集が楽すぎて、プレーンなテキストを書く際にもNeovimがないとつらいな、と思うようになっています...

最初はClaude Codeに任せきりだった設定ファイルも少しずつ読めるようになりました。

## Neovim を使うようになって変わったこと

VSCode を使っていたときには、文字を入力することにばかり頭がいっていた気がしますが、Neovim を使い始めてからはとにかくノーマルモードが楽しいです。
VSCode はコードレビューをするときなんかに開くことがありますが、Neovim のカーソル移動が楽すぎ&速すぎで、VSCode でちまちまやるのはちょっとイライラするようになりました。

あとは、いろんなコマンドの活用含め、「これちょっと面倒だな」と思うしきい値がいい意味で下がり、できる限り効率化したいという気持ちが高まっています。それはエディタに関することだけではなくほかの仕事に関しても「もっといいやり方がないか」を探索するクセがついてきました。これは Neovim を使い始めた副次的な効果のひとつですね。

## おわりにーNeovimはいいぞ

Neovim を使い始めてテキスト編集は圧倒的にはやくなったと思います。

なんといっても、マウスとキーボードの行き来をしなくてよいのが最高です。わたしは WezTerm を使っていて、ターミナル内で開発が簡潔するのがめちゃくちゃうれしい。

Neovim はいいぞ。
